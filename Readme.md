# Менеджер задач с Redux

## Описание проекта

Проект "Менеджер задач" - это приложение для управления задачами, переведенное с локального useState на глобальное управление состоянием с помощью Redux Toolkit.

## Как изменилась архитектура после внедрения Redux

### До внедрения Redux:
- Состояние задач хранилось в локальном useState в компоненте App
- Функции для изменения задач (addTask, toggleTask, deleteTask, clearCompletedTasks) передавались через пропсы во все дочерние компоненты
- Компоненты получали данные и функции через props drilling
- Логика работы с данными была распределена между компонентами

### После внедрения Redux:
- Состояние задач вынесено в глобальное Redux store
- Все компоненты получают данные через хук useSelector
- Все изменения данных происходят через dispatch экшенов
- Логика работы с данными централизована в tasksSlice
- Убрана передача функций через пропсы
- Добавлена папка features/tasks для группировки связанной логики

## Созданные редьюсеры

| Редьюсер | Назначение | Action Creator |
|----------|------------|----------------|
| addTask | Добавление новой задачи | addTask(task) |
| toggleTask | Переключение статуса задачи (выполнено/не выполнено) | toggleTask(taskId) |
| deleteTask | Удаление задачи по ID | deleteTask(taskId) |
| clearCompletedTasks | Очистка всех выполненных задач | clearCompletedTasks() |

## Созданные селекторы

| Селектор | Назначение |
|----------|------------|
| selectAllTasks | Получение всех задач из store |
| selectActiveTasks | Получение только активных (не выполненных) задач |
| selectCompletedTasks | Получение только выполненных задач |
| selectTasksStats | Получение статистики (всего, активных, выполненных) |
| selectFilteredTasks | Получение задач с фильтрацией по поиску и категории |
| selectTasksByCategory | Получение количества задач по категориям |
